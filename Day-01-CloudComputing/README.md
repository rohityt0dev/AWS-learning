# ☁️ What is Cloud Computing?

Cloud Computing is the on-demand delivery of computing resources such as servers, storage, databases, networking, software, and analytics over the Internet, allowing users to access resources whenever needed without owning or maintaining physical infrastructure.

Instead of purchasing expensive hardware, organizations rent computing resources from cloud providers and pay only for what they use.

---

## Traditional IT vs Cloud Computing

| Traditional IT | Cloud Computing |
|---------------|-----------------|
| Buy physical servers | Rent virtual servers |
| High upfront cost | Pay-as-you-go pricing |
| Manual hardware maintenance | Managed by cloud provider |
| Limited scalability | Scale resources in minutes |
| Long deployment time | Rapid provisioning |

# 🏗️ Types of Cloud Computing

Cloud services are generally divided into three categories:

## IaaS (Infrastructure as a Service)

- Virtual Machines
- Storage
- Networking

**Example:** Amazon EC2

---

## PaaS (Platform as a Service)

- Application deployment platform
- No server management

**Example:** AWS Elastic Beanstalk

---

## SaaS (Software as a Service)

- Ready-to-use software delivered over the internet

**Examples:**

- Gmail
- Microsoft 365
- Salesforce

---

# 🌐 Cloud Deployment Models

## Public Cloud

Infrastructure is owned and managed by a cloud provider.

Example:

- AWS
- Azure
- Google Cloud

---

## Private Cloud

Infrastructure is dedicated to a single organization.

---

## Hybrid Cloud

Combination of Public Cloud and Private Cloud.

---

## Multi-Cloud

Using services from multiple cloud providers.

Example:

AWS + Azure + Google Cloud

# 🌍 AWS Global Infrastructure

AWS operates one of the world's largest cloud infrastructures.

It consists of:

- Regions
- Availability Zones (AZs)
- Local Zones
- Edge Locations

### Region

A geographical area containing multiple Availability Zones.

Example:

- Mumbai (ap-south-1)
- Singapore (ap-southeast-1)

---

### Availability Zone (AZ)

One or more isolated data centers within a Region.

Benefits:

- High Availability
- Fault Tolerance

---

### Edge Location

Used by Amazon CloudFront to deliver content with low latency.

---

### Local Zone

Extends AWS services closer to users for latency-sensitive applications.

# 🔒 AWS Shared Responsibility Model

Security in the cloud is a shared responsibility between AWS and the customer.

| AWS is Responsible For | Customer is Responsible For |
|-------------------------|-----------------------------|
| Physical security | IAM users and permissions |
| Data center facilities | Operating system updates |
| Hardware | Application security |
| Networking infrastructure | Customer data |
| Global infrastructure | Security groups and firewalls |

### AWS Responsibility

- Physical servers
- Storage hardware
- Networking hardware
- Power
- Cooling
- Physical security

### Customer Responsibility

- User access management
- Data encryption
- Operating system patches
- Application security
- Backup configuration
