# 🔐 Lab: Enable Multi-Factor Authentication (MFA) for an IAM User

Multi-Factor Authentication (MFA) adds an extra layer of security to your AWS account. Instead of logging in with only a password, users must also provide a **one-time verification code** generated by an authenticator app or hardware device.

Even if an attacker knows your password, they **cannot access your account without the MFA code**.

---

# 🎯 Objective

- Understand AWS MFA
- Enable MFA for an IAM User
- Configure an Authenticator App
- Secure AWS Console login
- Remove or replace an MFA device

---

# 🛠️ Services Used

- AWS Identity and Access Management (IAM)
- Multi-Factor Authentication (MFA)

---

# 🏗️ Authentication Flow

```text
Username
    │
    ▼
Password
    │
    ▼
6-Digit MFA Code
    │
    ▼
AWS Login Successful
```

---

# 🔒 What is Multi-Factor Authentication (MFA)?

MFA is a security feature that requires **two or more forms of authentication** before granting access to an AWS account.

Authentication consists of:

- Something you know → **Password**
- Something you have → **Authenticator App or Hardware Security Key**

This significantly improves account security.

---

# 📱 Types of MFA Devices Supported by AWS

| MFA Type | Example | Recommended |
|----------|---------|-------------|
| Authenticator App | Google Authenticator, Microsoft Authenticator, Authy | ✅ Yes |
| Hardware Security Key | YubiKey (FIDO2/WebAuthn) | ✅ Yes |
| Hardware OTP Token | Supported hardware token | Used in some organizations |

# 🚀 Step 1: Sign in to AWS

1. Sign in to the **AWS Management Console**.
2. Search for:

```text
IAM
```

3. Open the **IAM** service.

---

# 👤 Step 2: Open Users

Navigate to:

```text
IAM
│
├── Dashboard
├── Users   ← Click here
├── User groups
├── Roles
└── Policies
```

Select the IAM user.

---

# 🔐 Step 3: Open Security Credentials

Select the user.

Open the:

```text
Security credentials
```

You will see sections such as:

- Console password
- Access keys
- Assigned MFA device

---

# 📲 Step 4: Assign an MFA Device

Under:

```text
Assigned MFA device
```

Click:

```text
Assign MFA device
```

---

# 📱 Step 5: Choose the MFA Device Type

For most users, select:

```text
Authenticator app
```

Click:

```text
Next
```

---

# 📥 Step 6: Install an Authenticator App

Install one of the following apps on your mobile device if you do not already have one:

- Google Authenticator
- Microsoft Authenticator
- Authy

---

# 📷 Step 7: Scan the QR Code

AWS displays:

- A QR Code
- Or a Secret Key

Open your authenticator app.

Tap:

```text
Add Account (+)
```

Scan the QR Code.

The app starts generating a new **6-digit code** every **30 seconds**.

Example:

```text
AWS Account

652481
```

---

# 🔢 Step 8: Enter Two Consecutive Codes

AWS asks for:

- MFA Code 1
- MFA Code 2

Enter:

1. The current 6-digit code.
2. Wait for the next code to generate.
3. Enter the new 6-digit code.

Click:

```text
Add MFA
```

---

# ✅ Step 9: Verify MFA Status

After successful configuration, you should see:

```text
Assigned MFA Device

Status:
Enabled
```

Your IAM user is now protected with Multi-Factor Authentication.

---

# 🔑 Logging in After MFA is Enabled

Each time you sign in:

1. Enter your username.
2. Enter your password.
3. Enter the current 6-digit MFA code from your authenticator app.

Authentication Flow:

```text
Username
    │
Password
    │
MFA Code
    │
AWS Login Successful
```

---

# 🔄 Remove or Replace an MFA Device

If you need to remove or replace an MFA device:

Navigate to:

```text
IAM
   ↓
Users
   ↓
Select User
   ↓
Security credentials
```

Under:

```text
Assigned MFA device
```

Choose:

```text
Manage
```

or

```text
Remove
```

(depending on the AWS Console version)

Follow the prompts to:

- Remove the existing MFA device
- Register a new MFA device

# ⚠️ Best Practices

- Enable MFA for all IAM users, especially administrators.
- Never share your MFA device or recovery codes.
- Keep a backup authentication method whenever possible.
- Use hardware security keys (FIDO2/WebAuthn) for maximum security.
- Always enable MFA on the AWS root account.
