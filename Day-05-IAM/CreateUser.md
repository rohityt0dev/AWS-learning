# 👤 Lab: Create an IAM User and Enable AWS Console Access

This lab demonstrates how to create a new **IAM (Identity and Access Management)** user and enable access to the AWS Management Console.

---

# 🎯 Objective

- Create a new IAM user
- Configure user details
- Enable AWS Management Console access
- Generate login credentials

---

# 🛠️ Services Used

- AWS Identity and Access Management (IAM)

---

# 🏗️ Architecture

```text
            AWS Account
                 │
                 ▼
          IAM Service
                 │
                 ▼
          Create IAM User
                 │
                 ▼
      Enable Console Access
                 │
                 ▼
      AWS Management Console
```

---

# 🚀 Step 1: Sign in to AWS

1. Sign in to the **AWS Management Console**.
2. In the search bar, type:

```text
IAM
```

3. Open the **IAM** service.

---

# 👤 Step 2: Create an IAM User

Navigate to:

```text
IAM
   ↓
Users
   ↓
Create user
```

---

# 📝 Step 3: Enter User Details

Enter the user name.

Example:

```text
rohit
```

Click:

```text
Next
```

---

# 🔐 Step 4: Set Permissions

Since you are only creating a user, you can:

- Leave the user without permissions for now.
- Attach IAM policies later.
- Add the user to an IAM Group later.

Click:

```text
Next
```

Review the configuration and click:

```text
Create user
```

The IAM user is now created.

---

# 🖥️ Step 5: Enable AWS Management Console Access

If the user needs to log in to the AWS Management Console:

Navigate to:

```text
IAM
   ↓
Users
   ↓
Select User (Example: rohit)
   ↓
Security credentials
```

Under **Console access**, click:

```text
Enable console access
```

Choose one of the following password options:

- **Autogenerated password**
- **Custom password**

> **Recommended:** Select:

```text
Users must create a new password at next sign-in
```

Click:

```text
Enable console access
```

---

# ✅ Expected Result

After completing this lab:

- A new IAM user is created.
- AWS Management Console access is enabled.
- The user receives a console login URL.
- The user can sign in using the assigned username and password.
- On the first login, the user is prompted to change their password.

---

# 📚 Learning Outcomes

After completing this lab, you will understand:

- How to create an IAM user
- How to enable AWS Management Console access
- How IAM users authenticate to AWS
- The difference between creating a user and assigning permissions
- Basic IAM user management

---

# ⚠️ Best Practices

- Follow the **Principle of Least Privilege** by granting only the permissions required.
- Use IAM Groups to manage permissions for multiple users.
- Require users to change their password at first sign-in.
- Enable Multi-Factor Authentication (MFA) for improved account security.
- Avoid using the AWS root account for everyday administrative tasks.
